# YandexHome — настройка

## Требования

- Доменное имя
- SSL-сертификат для HTTPS (рекомендуется Let's Encrypt)
- Навык «Умный дом» в [Яндекс.Диалогах](https://dialogs.yandex.ru/)

## Настройка навыка в Яндекс.Диалогах

1. Откройте https://dialogs.yandex.ru/ → «Создать навык» → «Создать диалог» → «Умный дом».
2. Заполните название (для приватного навыка не принципиально).
3. Укажите **Endpoint URL**: `https://ваш-домен/YandexHome`
4. Включите «Не показывать в каталоге», «Официальный навык» — нет.
5. Заполните остальные обязательные поля и загрузите иконку.
6. Вкладка **Связка аккаунтов** → «Создать»:
   - придумайте и запомните **идентификатор приложения** и **секрет**;
   - **URL авторизации**: `https://ваш-домен/YandexHome/auth/`
   - **URL для получения токена**: `https://ваш-домен/YandexHome/token/`
7. Сохраните навык → «На модерацию» (для приватного навыка проходит мгновенно) → «Опубликовать».
8. На вкладке **Тестирование** свяжите аккаунты, используя **Username** и **Password** из настроек модуля osysHome.

## Поля модуля

| Поле | Описание |
| --- | --- |
| **Username** | Имя пользователя для связки аккаунтов (User ID) |
| **Password** | Пароль для связки аккаунтов |
| **Client ID** | Идентификатор приложения из связки аккаунтов навыка |
| **Client secret** | Секрет приложения из связки аккаунтов навыка |
| **Client key** | OAuth-токен Яндекс.Диалогов для уведомлений о смене состояния |
| **Skill ID** | Идентификатор навыка (вкладка «Общие сведения» в консоли разработчика) |

## OAuth-токены для уведомлений

Для отправки изменений состояния устройств в Яндекс (свойство `reportable`) модуль обращается к API сервиса уведомлений:

```http
POST https://dialogs.yandex.net/api/v1/skills/{skill_id}/callback/state
Authorization: OAuth <токен>
```

Токен нужно получить **от аккаунта Яндекса, на котором создан навык**. Это не Client ID / Client secret из связки аккаунтов.

Документация Яндекса: https://yandex.ru/dev/dialogs/smart-home/doc/ru/reference-alerts/resources-alerts.html#oauth

### Client key — быстрый способ

1. Войдите в Яндекс под аккаунтом, на котором создан навык.
2. Откройте ссылку (кнопка с ключом в настройках модуля ведёт на неё же):

   https://oauth.yandex.ru/authorize?response_type=token&client_id=c473ca268cd749d3a8371351a8f2bcbd

3. Нажмите «Разрешить».
4. Скопируйте токен со страницы, которая откроется после авторизации.
5. Вставьте токен в поле **Client key**.

При истечении срока действия обновите токен вручную по той же ссылке. У публичного OAuth-приложения Яндекс.Диалогов нет `client_secret`, поэтому автоматическое обновление через refresh-токен недоступно (как в интеграции Home Assistant).

### Client key — через код авторизации (PKCE)

Альтернатива кнопке с ключом, если удобнее обменять код на токен в интерфейсе модуля:

1. Войдите в Яндекс под аккаунтом, на котором создан навык.
2. В настройках модуля нажмите ссылку **«через PKCE»** рядом с полем Client key — откроется страница Яндекса.
3. Нажмите «Разрешить» и скопируйте код со страницы Яндекса (можно вставить весь URL редиректа).
4. Вставьте код в поле **«Код авторизации»** и нажмите **«Обменять на токены»**.
5. Поле **Client key** заполнится автоматически; сохраните настройки.

При ошибке 401/403 модуль покажет уведомление со ссылкой на получение нового токена — обновите **Client key** вручную одним из способов выше.

## Добавление устройств

1. Откройте модуль YandexHome в админке osysHome.
2. Нажмите «Add device».
3. Укажите объект и свойства osysHome.
4. Настройте тип устройства и capabilities.
5. Устройство появится в приложении «Дом с Алисой» после обновления списка.
